## Technical Architecture — Client — Chi tiết sản phẩm

### Page Structure
- Zones: NAV (sticky top) · A — HeroImage · B — ProductInfo · C — ToppingSelector (conditional) · D — QtyStepper · E — CTAFooter (sticky bottom)
- Device target: mobile (375–420px primary; responsive up to desktop)
- Sticky zones: NAV (top-0 z-20) · E CTAFooter (bottom-0 z-20)
- Modals: none

### RBAC & Auth Rules

| Rule | Value |
|------|-------|
| **Route protection** | None (public page) — but requires valid `guestToken` in `useSettingsStore` |
| **Allowed roles** | Guest (customer) |
| **Auth state used** | `useSettingsStore.guestToken` — redirect to `/qr` if absent |
| **Conditional UI by role** | None — all zones visible to all guests |
| **Unauthorized redirect** | `router.replace('/qr')` when `guestToken` is missing |

### Tech Stack

```
React (Next.js 14 App Router)
├── State: Zustand (useCartStore write · useSettingsStore read) + no localStorage on this page
├── Data: TanStack Query (['products', id]) — prefetched ISR
├── Styling: Tailwind CSS (mobile-first, sticky top/bottom via `sticky` + z-index)
└── Types: TypeScript strict — ProductDetail · Topping · AddToCartPayload interfaces
```

### Key Implementation Patterns

1. **Component Architecture** — `page.tsx` is an RSC that prefetches the product query and wraps `ProductDetailClient` in `HydrationBoundary`. All interaction lives in `ProductDetailClient` ('use client').

2. **State Management**

```ts
// page-local only — no Zustand store needed for selection state
const [selectedToppingIds, setSelectedToppingIds] = useState<string[]>([])
const [quantity, setQuantity] = useState(1)

// Computed total (not stored)
const toppingTotal = product.toppings
  .filter(t => selectedToppingIds.includes(t.id))
  .reduce((sum, t) => sum + t.price, 0)   // price field, NOT price_delta
const unitPrice = product.price + toppingTotal
const total = unitPrice * quantity
```

3. **Data Fetching** — Single query `['products', id]` returns `ProductDetail` with `toppings[]` embedded. `staleTime: 300s` matches ISR `revalidate`. No second fetch for toppings.

4. **Performance** — ISR prefetch in `page.tsx` means zero loading flash on first paint for static product data. `ProductDetailSkeleton` handles the case when ISR cache misses (dynamic [id] not pre-generated).

5. **UX Enhancements** — Total price updates synchronously on every checkbox/stepper change (computed in render, no async). CTA always visible via sticky bottom. Back navigation via `router.back()` after add-to-cart.

6. **Edge Case Handling** — `is_available === false` → CTA disabled + Badge "Hết hàng". Image error → grey placeholder. `quantity` floored at 1 by `QuantityStepper min={1}`.

### Rendering Strategy

| Layer | What | Why |
|---|---|---|
| **ISR** (`revalidate: 300s`) | `['products', id]` prefetched in `page.tsx` | Product data is shared across all guests — not user-specific; safe to cache per `[id]` |
| **RSC** | `page.tsx` only — prefetch + HydrationBoundary | Server renders shell; client picks up hydrated query cache |
| **Client** (`'use client'`) | Zones NAV · A · B · C · D · E (all) | Zustand (useCartStore write) · localStorage (useSettingsStore) · interactive state |

> Gap: If the `[id]` was not pre-generated by `generateStaticParams`, ISR generates on first request — one cold-start skeleton visible. Not worth pre-generating all IDs for v1.

### Critical Implementation Notes
- Use `price` field on each `Topping` for the delta — **not** `price_delta`. This is explicit in the excalidraw comment.
- `router.back()` after `addItem` — return to menu. Do not `router.push('/menu')` which would break the browser history stack.
- Zone E must remain visible when the soft keyboard opens (avoid `position: fixed` antipatterns on iOS — use `sticky` with padding-bottom on the scroll container).
- `QuantityStepper` already handles `aria-label` and `role="spinbutton"` — do not reimplement.
