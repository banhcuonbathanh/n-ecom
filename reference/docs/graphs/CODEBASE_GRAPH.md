## flow — User Journey Sequence Diagrams

### Journey 1 — Customer: QR Scan → Order → Tracking

```mermaid
sequenceDiagram
    actor Customer
    participant QR as table/:tableId
    participant AuthBE as POST /auth/guest
    participant Menu as /menu page
    participant Cart as useCartStore
    participant Checkout as /checkout page
    participant OrderBE as POST /orders
    participant Track as /order/:id page
    participant SSE as SSE /orders/:id/events

    Customer->>QR: Scan QR code
    QR->>AuthBE: POST /auth/guest {qr_token}
    AuthBE-->>QR: access_token + table.id
    QR->>Menu: redirect /menu
    Menu->>Menu: GET /categories + /products + /combos
    Customer->>Menu: Browse, add to cart
    Menu->>Cart: addItem with toppings
    Customer->>Checkout: Tap Dat mon
    Checkout->>Cart: read items
    Checkout->>OrderBE: POST /orders {table_id, items}
    OrderBE-->>Checkout: order_id
    Checkout->>Track: redirect /order/:id
    Track->>SSE: open SSE stream via useOrderSSE
    SSE-->>Track: status events pending to ready
```

### Journey 2 — Staff: POS Order → KDS → Cashier Payment

```mermaid
sequenceDiagram
    actor Staff
    participant POS as /pos page
    participant OrderBE as POST /orders
    participant KDS as /kds page
    actor Chef
    participant WS as WS /ws/kds
    participant Cashier as /cashier/payment/:id
    participant PayBE as POST /payments

    Staff->>POS: Select products + set table
    POS->>OrderBE: POST /orders {items, table_id}
    OrderBE-->>WS: Redis pub new order event
    WS-->>KDS: push new order
    KDS-->>Chef: order on kitchen display
    Chef->>KDS: PATCH /orders/items/:id status served
    Chef->>KDS: PATCH /orders/:id/status ready
    Staff->>Cashier: Open /cashier/payment/:orderId
    Cashier->>Cashier: GET /orders/:id via useQuery
    Staff->>Cashier: Select payment method
    Cashier->>PayBE: POST /payments {order_id, method}
    PayBE-->>Cashier: payment_id + status completed
```

### Journey 3 — Manager: Live Floor Monitor (SSE + WS)

```mermaid
sequenceDiagram
    actor Manager
    participant Overview as /admin/overview page
    participant AdminSSE as useAdminSSE hook
    participant OverviewWS as useOverviewWS hook
    participant SSEBE as GET /sse/admin
    participant WSBE as WS /ws/kds
    participant Redis as Redis pub/sub
    participant OrderSvc as order_service

    Manager->>Overview: Open overview page
    Overview->>AdminSSE: mount connect SSE /sse/admin
    Overview->>OverviewWS: mount connect WS /ws/kds
    OrderSvc->>Redis: publish event on order change
    Redis-->>SSEBE: push event to admin channel
    SSEBE-->>AdminSSE: stream event {type, payload}
    AdminSSE-->>Overview: update live floor state
    WSBE-->>OverviewWS: push KDS updates
    OverviewWS-->>Overview: update prep panel state
    Manager->>Overview: Click Xac nhan on order
    Overview->>Overview: PATCH /orders/:id/status confirmed
```
