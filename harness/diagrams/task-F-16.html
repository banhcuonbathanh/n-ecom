02

The design's three sharpest ideas (kept as-is)

IdeaHow the schema does it
Price snapshotsThe client sends only ids + quantities. order_items stores name, unit_price, and toppings_snapshot frozen at order time — the customer can never set a price, and later menu edits never rewrite history.
Combo expansionA combo order becomes 1 header row (unit_price=0, the label) + N sub-item rows (combo_ref_id → header). The kitchen sees dishes, totals never double-count — enforced by a CHECK row-type matrix.
Derived facts get no columnItem status comes from qty_served vs quantity; ingredient status/expiry from stock + dates. Nothing stored that can drift from its source.
