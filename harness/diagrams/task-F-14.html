The decision: no folder moves

Every plausible rearrangement was weighed and rejected — the current five top-level areas each already have one clear job, and the paths are load-bearing:

  • reference/docs/… paths are cited across OVERALL_PLAN.md, BE_PLAYBOOK.md, DEVOPS.md, FE_STATE.md, four task pages and TASKS.md ACs — moving the corpus breaks dozens of citations for zero gain. It is a frozen north-star reference, not working material.
  • harness/diagrams/ pages link each other relatively (build-plan ↔ task pages) and are linked from VERIFICATION.md receipts — grouping task pages into a subfolder breaks the record.
  • Grouping the harness .md files into subfolders (e.g. design/) would break the CONTEXT_MAP routing table, README index, and every cross-link, to organize 17 files a human never lists — the agent navigates by CONTEXT_MAP, not by ls.

What actually drifted is the alignment layer: the indexes that promise to describe the repo no longer do. That is what this task repairs.
