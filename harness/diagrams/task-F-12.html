01

Why this task, why now

The owner asked: look at reference/docs/fe and consider making rules for this project. Gap analysis against the harness came back clean on architecture — state ownership (F-5), realtime and validation lessons (F-9 §6), tokens as a reference page (F-7) are covered. What has no home is the reference's convention layer: seven small "wrong vs right" rules that each killed hours in the old system. They are one-liners, so they extend FE_STATE.md §9 — the list every FE task already reads via CONTEXT_MAP — rather than becoming a new doc every session must also load.

One-fact-one-home: token values stay in design-system.html (F-7); behavioral defect lessons stay in OVERALL_PLAN.md §6; this task adds only the code-level rules neither file owns, and links instead of copying.

02

The 7 rules being adopted

#Rule (short form)Old-system pain it prevents
8Design tokens only — no raw hex, no raw palette classes (bg-orange-500) in any classNamedrifting shades of the same orange across pages; restyling required grep-and-pray
9Money renders through one formatVND() helper — never inline toLocaleString() or manual "₫" concatthree price formats on one screen
10FE types mirror the BE response exactly: field names from the DTO/curl receipt, never invented; IDs are stringFE invented base_price/slug/id:number — fields the API never sent; silent undefineds
11Derived display state is computed in a helper, never stored (e.g. item progress from served/total)stored copies of derivable values drifting from truth
12HTTP-method parity: FE call method must match the BE route registration; PATCH for partial updatesFE PATCH vs BE PUT → mystery 404s on a route that "exists"
13Staff/admin screens call their role-scoped (see-all) endpoints, never the public filtered onesadmin CRUD built on the public list → out-of-stock items invisible, unmanageable
14Asset paths from the API are relative; FE builds the full URL from env configabsolute URLs frozen into data, broken on every host change

Plus one environment fact (wrong home in FE rules, right home in ENVIRONMENT.md): Tailwind's JIT scans source at image build time — classes added after the Docker build are purged and render invisible. The fix is one line in the §Commands block: rebuild the FE image after adding new pages/classes.
