Decisions taken (defaults — say so if you want different)

  • The reference is the product spec, not the implementation spec. The old codebase's bugs and drift (documented per-page in *_BUGS.md / COMPARISON files) are treated as lessons, not requirements — the plan rebuilds on the clean F-5/F-6/F-8 architecture.
  • Existing phases are kept, not renumbered. C/CC/O/A rows stay; the plan adds new phases after them (staff, realtime, admin, ops) so no receipt or dependency link breaks.
  • 🔮 PLANNED reference pages stay planned. Pages the old system never built (introduction, storage, online-ordering entry) land in the latest phases.
  • Payment stays deferred per Session-0: cash/COD-style confirmation first; the cashier payment screen's gateway integration is a late phase.
