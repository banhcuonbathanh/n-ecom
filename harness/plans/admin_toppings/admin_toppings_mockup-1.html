One modal, two modes — the title and pre-fill are derived from whether an edit target is set. The status toggle works on create too; the reference hardcoded new toppings to “Có sẵn” and left the control lying. The duplicate-name error is a real server 409, not the reference's dead branch, and the modal stays open with nothing lost.

A real dialog, not a browser confirm(). The count comes from the row's own joined products[], so it is always truthful. Managers never see the Xóa button — delete is admin-only, and hiding it beats letting it 403.

Thử lại

Four named, mutually exclusive branches — loading, error, empty, ready. The reference collapsed loading into a single line of text (“Đang tải…”) that made the table jump when data arrived, and had no error branch at all, so a failed fetch looked exactly like “you have no toppings”.
