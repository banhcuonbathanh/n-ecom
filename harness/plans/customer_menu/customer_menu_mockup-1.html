What each region is

1
Photo banner header
components/menu/MenuHeader.tsx
196px gradient banner, Playfair serif title, table pill. Exactly ONE banner — the reference's duplicate RestaurantBanner is designed out (§6).
2
Search bar — reshapes the page
SearchBar.tsx · behavior §4.4-2
≥2 chars → hides nav/rail/combos, renders one flat client-side filtered list (or empty state). 1 char → "Nhập ít nhất 2 ký tự". Text lives in the URL ?q= (FE rule 2).
3
Category nav — the ONLY sticky element
MenuCategoryNav.tsx · §4.4-1
Anchor tabs, not filters: Tất cả · SUẤT · SUẤT TỰ TẠO · categories by sort_order. Tap smooth-scrolls; scroll spies the active tab (orange text + underline).
4
Favourites rail
FavouritesRail.tsx · favourites.store.ts
Device-local pinned set + custom suất + fav cards. Persisted Zustand store, FE-only — never crosses the wire (§4.2).
5
Combo section "SUẤT"
ComboSection.tsx · §4.4-3
Combo cards with nhân multi-select ("Nhân thịt" default, last can't be deselected). Names/prices joined from the products cache — combo payload is ids-only (§4.2, no duplicated payloads). Two buttons: "Món trong suất" expands the dish list in place (native toggle); "Xem chi tiết →" navigates to /menu/combo/:id (C-5, prefetch-on-hover §4.3).
6
Product sections + cards
ProductList.tsx / ProductCard.tsx · .menu-card + .stepper
Horizontal list rows (thumb left, body right — not a grid tile). Nhân single-select .opt-pill (first pre-selected), favourite heart (stopPropagation), the signature bordered .stepper, "HẾT" overlay when 86'd, fly-to-cart on add. Price 35.000 đ via formatVND(). Canh products never appear here (§4.4-4).
7
On-page cart — OrderSummary
OrderSummary.tsx · §4.4-4,5,6,7
Line editor + per-combo "Chi tiết" expander + "TỔNG SỐ MÓN" rollup + table pill. Canh chosen only via its stepper (2 rows: có rau / không rau). Note writes cart.orderNote, read by TableConfirmModal at POST time — the reference's 🔴 note-loss bug designed out.
8
Cart / checkout bar (inline, not floating)
CartBottomBar.tsx + TableConfirmModal.tsx
One "Thanh toán" pill with the item count as a badge inside it (no separate count button). Canh gate (§4.4-5): 0 bowls → pill dims but still fires → toast + auto-open cart + shake canh box. Table-bound → TableConfirmModal → POST /orders; no table → /checkout.
9
Floating support FAB — chat OR call staff
StaffContactFab.tsx (new — not yet in §4.1)
Small round button pinned bottom-right, floats over the menu on scroll. Tap expands two choices: 💬 Chat với NV and 📞 Gọi nhân viên — the guest picks either. Chat + staff-call belong to later phases (O/R realtime), so this is a placeholder surface; the plan doc doesn't list it yet — flagged below.

State B — search mode §4.4-2

  • Typing ≥2 chars reshapes the page: nav, rail and combo sections hide.
  • One flat, client-side filtered product list renders (or a "Không tìm thấy" empty state).
  • 1 char → inline hint "Nhập ít nhất 2 ký tự"; ✕ restores the sectioned menu.
  • Canh products stay filtered out of results too (isCanhProduct() — by CANH category_id, not by name; F34 · §4.4-4).
  • Query text is in the URL — shareable, survives reload (FE rule 2, not a store).

State C — loading & error tiers §4.3

  • First paint: RSC prefetches products/categories/combos → HydrationBoundary → zero-spinner; loading.tsx mirrors the real layout (banner + nav + rail + grid), never a centered spinner.
  • Five named branches: loading · error · empty · searching · data — the reference had 3 documented, 5 in code (§6).
  • Error: one inline slot "⚠ Kết nối mạng yếu" + Thử lại, covering all three queries — not per-section silence.
  • No cart mutation spinner — add-to-cart is pure client state; first server write is POST /orders (pessimistic).

State ownership at a glance §4.2

  • Server (products/categories/combos) → TanStack Query, staleTime 5 min = Redis TTL.
  • URL (?q=) → search text. Client (cart lines, tableId/Name, activeOrderId, orderNote) → persisted cart store. Client (favourites) → persisted favourites store.
  • Local (modal/expander open) → useState. Session (guest) → httpOnly cookie, no auth store.
  • Line ids encode the nhân variant so every filling combo is its own row and addItem dedups naturally; total()/itemCount() are selectors, never stored (FE rule 11).

The swap is one token change at build time — the customer shell re-binds the F-7 semantic token values at the (customer) layout scope rather than editing components (FE_PLAN §4.2), so flipping the hue touches one table.
